# Contributing

Contributions welcome. Before opening a PR:

1. Read [`AGENTS.md`](AGENTS.md) — project identity, scope, evidence and safety rules.
2. Read [`STATUS.md`](STATUS.md) — what's cleared, what isn't, what's deferred.
3. For examples: declare evidence mode (`live-source-backed`, `user-provided sources`, `illustrative source packet`, `reasoning-only`). Do not fabricate citations, sanctions designations, vessel names, IMO numbers, prices, or dates.
4. For docs: keep mechanism-first reasoning. Avoid generic Middle East commentary.
5. For external reviews of examples or rubric applications: include your name/role and attribution. Anonymous reviews don't count toward Bar 2.

Small, additive PRs preferred. Do not rewrite the project structure unless explicitly requested in an issue.
